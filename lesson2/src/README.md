# Lesson 2 Lab – Manual Test Suite Design

## Goal

Design a **manual test suite** for a method and reflect on how you chose your tests.

This lab focuses on:
- Selecting inputs
- Defining expected outputs (your oracle)
- Writing test cases in a structured way

You are **not required** to write automated tests.

---

## What to Test

1. Open `planning_poker.py`
2. Locate the method `identify_extremes(estimates)`

---

## What to Do (Step-by-step)

### Step 1 — Read the method (do not run it yet)
- Understand what the method does
- Identify inputs and outputs

### Step 2 — Clarify the intended behavior
Write down (in your own words):
- What should the method return?
- What kind of input does it expect?

### Step 3 — Design the test cases
- Use the template in `test_suite_template.md`
- Write **at least 3 test cases**

### Step 4 — Prepare to discuss
Be ready to explain:
- Why you chose those tests
- What your tests reveal about the method
- How confident you are after testing it

Submit:
- `YOUR_NAME_test_suite_template.md` filled in
